# GAME_DESIGN — 3D Tennis ゲーム仕様書

## 1. コンセプト

ブラウザで手軽に遊べる3Dテニス。アーケード的な操作の軽快さと、
**「配球・リスク管理・体力管理」の3軸の戦略性**を両立する。
単にボールを打ち返すだけでは勝てず、相手を走らせてオープンコートを作り、
リスクの高い決め球をどこで使うかの判断が勝敗を分ける。

- シングルプレイヤー(プレイヤー vs AI)
- 視点はプレイヤー後方の三人称カメラ
- 外部アセットなしのローポリ表現(プリミティブジオメトリ+合成音)

## 2. ルールとマッチ設定

実テニスの簡略版を採用する。

- **ポイント**: 0 → 15 → 30 → 40 → ゲーム。40-40 はデュース。
  デュース後はアドバンテージ → ゲーム(ノーアドなし)。
- **マッチ**: タイトル画面で先取ゲーム数を選択 — **1 / 2 / 4 ゲーム先取**
  (デフォルト 2)。セットはなし。タイブレークなし。
- **サーブ権**: 1ゲームごとに交代。最初はプレイヤー。
- **サーブ位置**: ポイント合計が偶数ならデュースサイド(右)、奇数ならアドサイド(左)。
  対角のサービスボックスに入れる。フォルト2回でダブルフォルト(失点)。
- **コートチェンジはなし**(プレイヤーは常に手前側。簡略化)。
- **レット(ネットイン・サーブ)はなし**: ネットに触れてもボックスに入ればインプレー。
- **イン/アウト判定**: 打球が相手コート(シングルスライン内、ライン上はイン)に
  1バウンド目で入らなければ打った側の失点。自陣でバウンドした(ネットを越えな
  かった)場合も失点。2バウンドされたら受け手の失点。

## 3. 操作方法(キーボード)

| 入力 | 動作 |
|---|---|
| W / A / S / D(または矢印キー) | 移動 |
| Shift(押しっぱなし) | スプリント(高速移動・スタミナ消費) |
| J | フラットショット |
| K | トップスピン |
| L | スライス |
| U | ロブ |
| I | ドロップショット |
| 移動キー + ショットキー | コースの打ち分け(A/D=左右、W=深く、S=浅く) |
| ショットキー長押し | チャージ(§4.4)。溜めるほど強いが、溜めすぎはリスク |
| J / K / L(サーブ時) | サーブの種類を選択(J=フラット / K=キック / L=スライス、§5.1) |
| Space(サーブ時) | 押すとパワーメーター開始、離した瞬間の値で威力決定 |
| Esc | ポーズ / タイトルへ |

ショットキーを**押しっぱなしにするとチャージ**し、ボールがリーチに入った瞬間に
自動で打球する(§4.4)。ボールが既にリーチ内にあるときに押せば即打(チャージ小)。
ボールが届かないまま離すと 0.25 秒の再チャージ不可時間が生じる。
操作説明はプレイ中も画面端に常時表示する(§9)。

## 4. ショットシステム

### 4.1 ショット5種の性質

| 種類 | 弾道 | 速度 | スピン | バウンド後 | 主な用途 / リスク |
|---|---|---|---|---|---|
| フラット | 低い | 最速 | なし | 普通 | 決め球。ネット・アウトのリスク最大 |
| トップスピン | 高め | 中 | 順回転(落ちる) | 高く速く跳ねる | 安定したラリーの主軸。リスク小 |
| スライス | 低い | 遅 | 逆回転(伸びる) | 低く滑る | 時間を作る・リズムを崩す。浮くと打ち込まれる |
| ロブ | 非常に高い | 遅 | 軽い順回転 | 深く落ちる | ネットに出た相手の頭上を抜く |
| ドロップ | 低い | 最遅 | 逆回転 | ほぼ跳ねない | ベースラインに張り付く相手を前に走らせる。甘いと拾われ反撃される |

### 4.2 ショット品質(クオリティ)

すべてのショットは打球時に品質 q ∈ [0.35, 1.0] を持ち、以下で決まる:

```
q = clamp( 距離係数 × スタミナ係数 − スプリントペナルティ, 0.35, 1.0 )
  距離係数        : ボールとの距離 0.9m 以内で 1.0、2.0m(リーチ限界)で 0.35 へ線形減衰
  スタミナ係数    : スタミナ 30% 以上で 1.0、0% で 0.6 へ線形減衰
  スプリントペナルティ: スプリント中の打球は −0.15
```

品質の効果:
- **狙いの誤差**: 着地目標点に半径 `(1−q) × 2.2m` の一様ノイズが乗る。
  低品質でライン際を狙うとアウトしやすい。
- **威力**: ボール初速に `0.75 + 0.25q` を乗算。
- **フラットのネットリスク**: フラットは弾道が低いため、低品質だと
  ネット直撃の確率が上がる(誤差ノイズが高さ方向にも乗る)。

→ 戦略: **良い体勢(ボールの正面に早く入る)で打つほど強気の配球ができる**。
走らされた状態では安全なトップスピンやスライスで凌ぐのが正解になる。

### 4.3 コースの打ち分け

ショットの基準着地点はショット種ごとに定義し(相手コート中央〜深め)、
打球時に押している移動キーでオフセットする:

- A / D: 左右に ±2.6m(シングルスライン際まで)
- W: 2.0m 深く(ベースライン際)/ S: 2.0m 浅く
- 目標点はコート内側 0.3m のマージンにクランプされるが、
  品質ノイズで外に出ることはある(=ライン際を狙うほどリスク)

### 4.4 チャージショット

ショットキーの長押しで力を溜める:

- チャージ量 c は 0 から 0.8 秒で 1.0 に達し、さらに最大 1.25 まで増加する
  (1.0 超は**オーバーチャージ**)。
- **威力**: ボール初速に `0.85 + 0.40·min(c, 1)` を乗算(品質の威力係数と併用)。
- **オーバーチャージのリスク**: c > 1 では狙い誤差が `(c−1) × 2.8m` 追加され、
  ネット越えマージンも縮小する。最大威力を狙うほどアウト/ネットが増える。
- **機動力の代償**: チャージ中は移動速度が 45% に低下する。
  早めに構えるほど強く打てるが、ボールの正面に入る調整が利かなくなる。
- **インパクト前後の硬直**: 打球の瞬間から 0.35 秒間、移動はほぼ停止する
  (12%)。打ちながら走り抜ける不自然な動きはできない。

→ 戦略: 「早く構えて強打」か「ぎりぎりまで動いて確実に返す」かの選択が
毎球発生する。チャンスボールでのフルチャージはリターンが大きいが、
溜めすぎ(オーバーチャージ)は自滅につながる。

### 4.5 打点・コート位置・球威による打球変化(戦略性の核)

「コートのどこで・どの高さで・どんな勢いの球を打つか」で打球が大きく変わる。
基準ショット(§4.1)に対し、以下の文脈修飾が掛かる。これにより
**ただ打ち返すだけでは安定せず、状況に応じた選択が要る**。

打点の高さ `h`(ボールの y)からレバレッジ `lev = clamp((h−0.9)/0.9, −1, +1)` を定義
(低い=−、通常=0、高い=+)。コート位置はネットからの距離 `depth = |hitPos.z|`、
「前寄り(forecourt)」は `depth < サービスライン(6.4m)`。相手球の勢い `vIn`。

- **スマッシュ(高い打点 × 前寄り × フラット)**: 打点が高く(≥1.7m)前寄り
  (ネットから 8.5m 以内)でフラットを打つと**スマッシュ**になり、初速が
  大幅に上がり(基準 42 m/s、チャージでさらに +最大45%)上から叩き込む。
  品質低下の影響も小さい。→ 高く弾む球や短いロブを前で仕留める決め球。
  ベースライン深くでは(前寄りでないため)成立せず、ただの高打点フラット。
- **低い打点でのパワーはアウト/ネットになりやすい**: 低い打点(lev<0)で
  強打(チャージ/クリーンヒット)すると、持ち上げて入れる必要があるのに
  パワーが乗るため**深さがオーバーして高確率でアウト**、フラットだと逆に
  ネットにも掛かりやすい。前寄りだとこのリスクはさらに増幅(後ろが短い)。
  → 低い球は無理に叩かず、トップスピン/スライスで安全に返すのが正解。
- **高い打点のトップスピンは角度をつけられる**: 高い打点(lev>0)のトップスピンは
  横の狙い幅が最大 +85% 広がり、回転も増えて沈むため、**鋭いクロスの
  アングルショット**が決まる(低い打点では角度をつけにくい)。
  → 高く跳ねた球を高い打点で捉えてオープンコートを作る。
- **前寄りフラットの深さリスク**: 前に詰めて低〜中打点でフラットを強打すると、
  後ろのコートが短いためアウトしやすい。前で安全に攻めるには高い打点(叩く)か
  角度・緩急が必要。
- **相手球の勢い(球威)**:
  - **カウンター/リダイレクト**: 速い球はフラット/スライスで合わせると相手の
    勢いを利用でき、自分の労力以上の速球で返る(球速の約30%上乗せ。
    トップスピンは約12%)。速い展開での「合わせる」技術。
  - **速球の制御難**: `vIn` が 17 m/s を超えると、体勢が悪い(品質が低い)ほど
    狙いが乱れる。速い球を走らされながら強打すると自滅しやすい。
  - **緩急の吸収難**: ドロップ/ロブのタッチショットは速い球からは打ちにくく
    (誤差増)、緩い球から打つのが成功率が高い。

→ 設計意図: これらは加算的・連続的に効くため、通常のベースラインラリー
(中くらいの打点・中庸な球威)はほぼ従来どおり安定するが、**極端な状況
(ネット際・高い/低い打点・速い球・走らされ)では結果が大きく変わる**。
「いつ前に出るか」「高い球をどう使うか」「速い球を合わせるか凌ぐか」という
判断が常に発生する。

### 4.6 速いボールの返球(ミート/差し込まれ)

スマッシュのような速いボールは、ボールが届く位置にいても**芯で捉えてミート
するのが難しい**。準備(チャージ)が浅いと差し込まれ、しっかり返せずに
**山なりの弱い返球(チャンスボール)**になる。これによりスマッシュ等の
強打に戦術的価値が生まれる(強く打つほど相手の返球が甘くなりやすい)。

現実のテニスのセオリーに基づくショット種ごとの「速球への強さ」:

- **スライス(ブロック)が最も強い**: 短いテイクバックで球威を利用して
  返せる。速いサーブのリターンでプロが多用する基本。低く滑って返る。
- **フラットは中間**: 球威を合わせて返せるが、タイミングがシビア。
- **トップスピンは最も弱い**: フルスイングと精密なタイミングが必要なため、
  速球に差し込まれると最も崩れやすい(ただし**十分に準備=チャージ**できれば
  強烈な返球も可能)。
- ロブ/ドロップのタッチショットは速球からは最も難しい(§4.5 とも整合)。

差し込まれ度 `mishit`(0〜1)は「相手球速がしきい値(26 m/s)をどれだけ超えたか
× ショット種の弱さ × (1−チャージによる準備) × 体勢(品質)」で決まる。
mishit が大きいほど、返球は **遅く・高く・浅く**(山なりのsitter)なり、
回転も失う。チャージで準備し、スライスで合わせれば速球も deep に返せるが、
不用意なトップスピンの当てるだけの返球は浮いて絶好球になる。

→ 戦術: 高い球を前で**スマッシュ**(§4.5)で叩き込めば、相手は差し込まれて
甘い返球になりやすく、次球で決められる。逆に、相手の強打を受けるときは
**早めにチャージしてスライスでブロック**するのが安全。通常のラリー球速
(〜25 m/s)では差し込まれは起きず、従来どおり打ち合える。

## 5. サーブ

### 5.1 サーブの種類

サーブは方向(A/D)に加え、**種類を選べる**(サーブ前に J/K/L で選択。
デフォルトはフラット)。現実のテニスの3大サーブを再現する:

| 種類 | キー | 速度 | スピン | バウンド | 特徴・戦術 |
|---|---|---|---|---|---|
| フラット | J | 最速 | ほぼ無回転 | 低く直進 | 最速だがマージンが小さくフォルトしやすい。1stの決め球 |
| スライス | L | 中 | サイドスピン | 低く滑り横へ曲がる | 相手をワイドに追い出しコートを開ける。安全め |
| キック | K | 遅 | 重い順回転+横 | 高く弾んで跳ね上がる | ネット越えマージンが大きく安全。高く弾んで相手を後方/上へ押す。2ndの主軸 |

→ 戦術: 1stはフラット/スライスで攻め、フォルトしたら2ndはキックで安全に
入れる、という現実の駆け引きを再現。キックサーブは高く弾むため、相手は
高い打点で returns することになり(§4.5)、差し込まれや体勢の崩れを誘える。

### 5.2 サーブの操作と威力

1. サーブサイドの定位置に配置された後、**サーブを打つサイドの範囲内で
   立ち位置を移動できる**(x はセンターマーク±0.25m からシングルスライン、
   z はベースライン後方 0.2〜2.5m)。ワイドに寄って角度をつけるなどの
   駆け引きが可能。`Space` 押下でパワーメーターが往復開始
   (0→1→0、周期 1.2 秒の三角波)。メーター中は移動できない。
2. 離した瞬間の値 p が威力。A/D でボックス内の左右コースを選択。
3. 判定:
   - `p ∈ [0.70, 0.88]` — スイートゾーン。速く正確なサーブ。
   - `p > 0.88` — オーバーパワー。速いが誤差大、フォルト率高。
   - `p < 0.70` — 威力・誤差ともに p に比例(安全だが遅い)。
4. フォルト時の2ndサーブも操作は同じ。ダブルフォルトは失点。

→ 戦略: 1stは強気・2ndは安全という実テニスの駆け引きを再現。

## 6. スタミナ

- 最大 100。HUD にバー表示(プレイヤー・AI 両方)。
- スプリント中: 18/秒 消費。歩行・静止中: 6/秒 回復。
- ポイント間: 40 回復。ゲーム間: 全回復。
- 効果: 30% 未満でショット品質が落ち始め(§4.2)、0% でスプリント不可。

→ 戦略: 序盤から全力で走り回ると、長いラリーの終盤で精度が崩壊する。
相手を走らせて自分は走らされない配球が長期的に効く。

## 7. AI

### 7.1 行動モデル

- ラリー中はボールの**着地予測点**へ移動(反応遅延後)。
  自分が打たないときはホームポジション(ベースライン中央寄り)へ戻る。
- **アウトの見送り**: 相手の打球の着地予測が自陣コート外(ベースライン奥/サイド外)の
  ときは打たずに見送る。明らかにアウト(ライン外 0.6m 超)なら高確率で見送り、
  きわどいもの(ライン際)は確率的に見送る/打つ。難易度が高いほど見送り精度が高い
  (Hard はほぼ確実に明らかなアウトを見送る)。コート内に入る球は必ず打つ。
  見送ると判断した球は追わずホームへ戻る。
- ショット選択は以下を重み付けで評価:
  - **オープンコート**: プレイヤー位置から遠いサイドを狙う(基本)
  - **体勢**: 走らされているときは安全なショット(トップスピン/スライス/ロブ)
  - **プレイヤーが前に出たら**ロブ/パッシング(サイド強打)
  - **プレイヤーが深く下がったら**ドロップショット
- AI にも品質システムとスタミナが同じルールで適用される。

### 7.2 難易度(タイトル画面で選択)

| パラメータ | Easy | Normal | Hard |
|---|---|---|---|
| 反応遅延 | 0.42秒 | 0.28秒 | 0.16秒 |
| 移動速度係数 | 0.82 | 0.95 | 1.05 |
| 狙いの誤差(追加ノイズ) | 大 | 中 | 小 |
| 強気な配球の頻度 | 低 | 中 | 高 |
| 凡ミス率(時々ある明らかなミス) | 8% | 4% | 1.5% |
| 明らかなアウトの見送り率 | 82% | 92% | 98% |
| きわどいアウトの見送り率 | 35% | 45% | 55% |

→ どの難易度でも「左右に振ってから逆を突く」「前後の揺さぶり」という
**人間的な攻略法が通る**ように、AI は予測点ベースで動き先読みはしない。

## 8. カメラ・演出

- プレイヤー後方上空からの追従カメラ(位置・注視点ともに緩い lerp)。
  ボールとプレイヤーの中間点を注視。
- **ボールの視認性**(最優先。ボール自体は実寸の 1.35 倍に留め、補助表現で追いやすくする):
  - 発光ハロー(加算合成の billboard)でボールを目立たせる
  - 長く太い残像トレイル(速度が速いほど顕著)
  - **着地予測マーカー**: 飛行中のボールの着地点にリングを表示(脈動アニメ)
  - ボール真下のグラウンドマーカー(擬似丸影を強調)
  - バウンド時にリング状のダストエフェクト
- **プレイヤーモデル**: 頭・胴体・両腕・両脚を持つ人型。利き手(右)に
  ラケットを持ち、フォアハンド/バックハンドのスイングが見分けられる。
  チャージ中はボール側へテイクバック(チャージ量で引きの深さが変わる)。
- ポイント決着時: スロー演出はなし。バナー表示(例「15 - 30」「Game!」)
  → 1.8 秒後に次のサーブへ。

## 9. UI

- **タイトル画面**: 難易度選択、先取ゲーム数選択、操作説明、開始ボタン。
- **HUD**(プレイ中)— フォントは読みやすさ優先で大きめに:
  - 上部中央: スコアボード(ゲーム数 + 現在ポイント、サーブ権マーク)
  - 下部左右: プレイヤー / AI のスタミナバー
  - **画面端に操作説明を常時表示**(コンパクトなキー一覧。プレイの邪魔に
    ならない半透明パネル)
  - チャージバー(チャージ中のみ。オーバーチャージ域は警告色)
  - サーブ時: パワーメーター(スイートゾーンを色分け)
  - ポイント決着バナー(得点理由つき: 「Winner!」「Out」「Net」「Double Fault」)
- **マッチ終了画面**: 勝敗、スタッツ(ウィナー数・エラー数)、再戦/タイトルへ。

## 10. オーディオ

WebAudio による合成のみ(外部ファイルなし):
打球音(ノイズバースト、ショット種で音色差)、バウンド音、ネット音、
ポイント決着の歓声(フィルタ付きノイズ)、UI クリック音。初回操作で AudioContext を resume。

## 11. 戦略性の設計意図(まとめ)

1. **配球**: 5種のショット × 左右前後の打ち分けで相手を動かし、オープンコートを作る。
2. **リスク管理**: 品質システムにより「体勢が悪いのに強打」は自滅する。
   ライン際を狙うか安全に返すかの判断が常に発生する。
3. **体力管理**: スプリントは有限資源。長いマッチほど配分が効く。
4. **サーブの駆け引き**: パワーメーターによる 1st/2nd のリスク選択。
5. **AI の隙**: AI は正直に予測点へ走るので、振り回し・逆を突く・前後の
   揺さぶりという現実のテニスのセオリーがそのまま攻略法になる。
