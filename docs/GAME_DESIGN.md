# GAME_DESIGN — 3D Tennis ゲーム仕様書

## 1. コンセプト

ブラウザで手軽に遊べる3Dテニス。アーケード的な操作の軽快さと、
**「配球・リスク管理・体力管理」の3軸の戦略性**を両立する。
単にボールを打ち返すだけでは勝てず、相手を走らせてオープンコートを作り、
リスクの高い決め球をどこで使うかの判断が勝敗を分ける。

- シングルプレイヤー(プレイヤー vs AI)
- 視点はプレイヤー後方の三人称カメラ
- 外部アセットなしのローポリ表現(プリミティブジオメトリ+合成音)

## 2. ルールとマッチ設定

実テニスの簡略版を採用する。

- **ポイント**: 0 → 15 → 30 → 40 → ゲーム。40-40 はデュース。
  デュース後はアドバンテージ → ゲーム(ノーアドなし)。
- **マッチ**: タイトル画面で先取ゲーム数を選択 — **1 / 2 / 4 ゲーム先取**
  (デフォルト 2)。セットはなし。タイブレークなし。
- **サーブ権**: 1ゲームごとに交代。最初はプレイヤー。
- **サーブ位置**: ポイント合計が偶数ならデュースサイド(右)、奇数ならアドサイド(左)。
  対角のサービスボックスに入れる。フォルト2回でダブルフォルト(失点)。
- **コートチェンジはなし**(プレイヤーは常に手前側。簡略化)。
- **レット(ネットイン・サーブ)はなし**: ネットに触れてもボックスに入ればインプレー。
- **イン/アウト判定**: 打球が相手コート(シングルスライン内、ライン上はイン)に
  1バウンド目で入らなければ打った側の失点。自陣でバウンドした(ネットを越えな
  かった)場合も失点。2バウンドされたら受け手の失点。

## 3. 操作方法(キーボード)

| 入力 | 動作 |
|---|---|
| W / A / S / D(または矢印キー) | 移動 |
| Shift(押しっぱなし) | スプリント(高速移動・スタミナ消費) |
| J | フラットショット |
| K | トップスピン |
| L | スライス |
| U | ロブ |
| I | ドロップショット |
| 移動キー + ショットキー | コースの打ち分け(A/D=左右、W=深く、S=浅く) |
| ショットキー長押し→離す | 長押しでチャージ、**離した瞬間に打球**(§4.4)。芯で離すとジャスト |
| J / K / L(サーブ時) | サーブの種類を選択(J=フラット / K=キック / L=スライス、§5.1) |
| Space(サーブ時) | 押すとパワーメーター開始、離した瞬間の値で威力決定 |
| Esc | プレイ中のポーズ / 解除(ポーズ中に「ゲーム終了」でタイトルへ) |
| 0 | デバッグ表示のトグル(AI判断ログ。ARCHITECTURE §17) |

ショットキーを**押しっぱなしにするとチャージ**し、**離した瞬間にリーチ内なら打球**する(§4.4)。
離したときボールが芯(スイートゾーン)にあれば**ジャストミート**(§4.4.1)。離さずにいても
ボールがリーチを抜ける直前に**セーフティ打球**(ジャストなし)するので返球は途切れない。
ボールがリーチ外のときに離すと空振りになり 0.25 秒の再チャージ不可時間が生じる。
操作説明はプレイ中も画面端に常時表示する(§9)。

## 4. ショットシステム

### 4.1 ショット5種の性質

| 種類 | 弾道 | 速度 | スピン | バウンド後 | 主な用途 / リスク |
|---|---|---|---|---|---|
| フラット | 低い | 最速 | なし | 普通 | 決め球。ネット・アウトのリスク最大 |
| トップスピン | 高め | 中 | 順回転(落ちる) | 高く速く跳ねる | 安定したラリーの主軸。リスク小 |
| スライス | 低い | 遅 | 逆回転(伸びる) | 低く滑る | 時間を作る・リズムを崩す。浮くと打ち込まれる |
| ロブ | 非常に高い | 遅 | 軽い順回転 | 深く落ちる | ネットに出た相手の頭上を抜く |
| ドロップ | 低い | 最遅 | 逆回転 | ほぼ跳ねない | ベースラインに張り付く相手を前に走らせる。甘いと拾われ反撃される |

### 4.2 ショット品質(クオリティ)

すべてのショットは打球時に品質 q ∈ [0.35, 1.0] を持ち、以下で決まる:

```
q = clamp( 距離係数 − スプリントペナルティ, 0.35, 1.0 )
  距離係数        : ボールとの距離 0.9m 以内で 1.0、2.0m(リーチ限界)で 0.35 へ線形減衰
  スプリントペナルティ: スプリント中の打球は −0.15
```

スタミナ残量による品質低下は**廃止**。スタミナ切れは「強打不可・スプリント不可」という
能力ゲートで表現する(§6 参照)。スタミナが少なくても打球品質自体は落ちない。

品質の効果:
- **狙いの誤差**: 着地目標点に半径 `(1−q) × 2.2m` の一様ノイズが乗る。
  低品質でライン際を狙うとアウトしやすい。
- **威力**: ボール初速に `0.75 + 0.25q` を乗算。
- **フラットのネットリスク**: フラットは弾道が低いため、低品質だと
  ネット直撃の確率が上がる(誤差ノイズが高さ方向にも乗る)。

→ 戦略: **良い体勢(ボールの正面に早く入る)で打つほど強気の配球ができる**。
走らされた状態では安全なトップスピンやスライスで凌ぐのが正解になる。

### 4.3 コースの打ち分け

ショットの基準着地点はショット種ごとに定義し(相手コート中央〜深め)、
打球時に押している移動キーでオフセットする:

- A / D: 左右に ±2.6m(シングルスライン際まで)
- W: 2.0m 深く(ベースライン際)/ S: 2.0m 浅く
- 目標点はコート内側 0.3m のマージンにクランプされるが、
  品質ノイズで外に出ることはある(=ライン際を狙うほどリスク)

### 4.4 チャージショット(リリース方式)

ショットキーの**長押しで力を溜め、離した瞬間に打球**する:

- チャージ量 c は 0 から 0.8 秒で 1.0 に達し、さらに最大 1.25 まで増加する
  (1.0 超は**オーバーチャージ**)。**離した瞬間**のチャージ量で威力が決まる。
- **打球タイミング**: チャージキーを離した瞬間、ボールがリーチ内なら打球。**離したとき
  ボールが芯(スイートゾーン)にあればジャストミート**(§4.4.1)。離さずにいても、ボールが
  リーチを抜ける直前に**セーフティ打球**(ジャストなし)するので返球は途切れない。
  「抜ける直前」は2軸で判定: **水平に遠ざかる**(横に通過する通常球)か、**下降して低くなる**
  (`SAFETY_DROP_Y` 以下。真下に落ちる山なり/ロブ/スマッシュ。これが無いと落下球を空振りする)。
  ただし**ボールが水平に明確に近づいている間**(closingRate > `SAFETY_APPROACH_RATE`)は垂直
  セーフティを出さず、芯に入れて just を狙う余地を残す(低い球をノーバウンドで取りに行くとき、
  まだ遠い=リーチ端で即セーフティが出て just が出せなかったのを防ぐ)。
  リーチ外で離すと**空振り**(0.25 秒の再チャージ不可)。
- **威力**: ボール初速に `0.85 + 0.40·min(c, 1)` を乗算(品質の威力係数と併用)。
- **オーバーチャージのリスク**: c > 1 では狙い誤差が `(c−1) × 2.8m` 追加され、
  ネット越えマージンも縮小する。最大威力を狙うほどアウト/ネットが増える。
- **機動力の代償**: チャージ中は移動速度が 45% に低下する。
  早めに構えるほど強く打てるが、ボールの正面に入る調整が利かなくなる。
- **インパクト前後の硬直**: 打球の瞬間から 0.35 秒間、移動はほぼ停止する
  (12%)。打ちながら走り抜ける不自然な動きはできない。

→ 戦略: 「早く構えて強打」か「ぎりぎりまで動いて確実に返す」かに加え、**離すタイミング**で
芯を捉える(ジャスト)読みが乗る。溜めすぎ(オーバーチャージ)は自滅につながる。

#### 4.4.1 ジャストミート(離すタイミングの上積み)

**離した瞬間にボールが芯(スイートゾーン=打点距離 `JUST_SWEET_DIST`=1.0m 以内)にあると
「ジャストミート」**になり、控えめなボーナスがつく(IMPROVEMENTS §6.1.1)。芯を外しても通常
打球で**ペナルティはない**(腕前の天井を上げる任意要素)。

- **入力**: チャージ(長押し)→ ボールが芯に来た瞬間に**離す**。芯にいる間はずっと狙えるが、
  **速球ほどスイートゾーンの通過が速い**ため自然に難しくなる(本作の速球制御難 §4.6 と整合)。
  離さずにいるとスイートゾーンを抜けた瞬間にセーフティ打球(ジャストなし)になる。
- **ボーナス**(いずれも控えめ): 初速 ×1.08 / 狙い誤差 ×0.6(正確に)/ 回転 ×1.12。
- **演出**: 接触点に金白の発光リング+スパーク、打球は飛行中ハロー/トレイルが金色、打球音に
  澄んだベル倍音(オーディオ §10 `just`)。決定論は保たれる(入力タイミング依存のみ、乱数なし)。
- **タイミングヒント**: プレイヤー足元に**収束リング**を表示。ボールが近づくほど外周リングが縮み、
  **芯(スイートゾーン)に入ると金色+脈動で「今離す」**を示す。打てるまでの時間はボールの
  前方シミュレート(`BallSim.predictReach`)で求め、接触の約 0.6 秒前から出る。
- **デバッグ**: `?debug` 時、毎打のジャスト判定(`OK`/`--`・打点距離 hDist・芯閾値 sweet・
  球速・品質・トリガ `release`/`safety`)をログ出力。成立か・芯にどれだけ近かったかを確認できる。
  さらに**空振り**(リリースしたがリーチ外: `whiff(理由) ball=.. me=.. hDist=../.. y=../..`)と
  **見送り/未到達**(返せず終わった相手球の最接近: `miss(理由) closest hDist=.. ball=.. me=..`、
  理由= `TOO-FAR`/`TOO-HIGH`/`in-reach-未打`)もログ化。何がズレて取れないかを分析できる。

### 4.5 打点・コート位置・球威による打球変化(戦略性の核)

「コートのどこで・どの高さで・どんな勢いの球を打つか」で打球が大きく変わる。
基準ショット(§4.1)に対し、以下の文脈修飾が掛かる。これにより
**ただ打ち返すだけでは安定せず、状況に応じた選択が要る**。

打点の高さ `h`(ボールの y)からレバレッジ `lev = clamp((h−0.9)/0.9, −1, +1)` を定義
(低い=−、通常=0、高い=+)。コート位置はネットからの距離 `depth = |hitPos.z|`、
「前寄り(forecourt)」は `depth < サービスライン(6.4m)`。相手球の勢い `vIn`。

- **スマッシュ(高い打点 × 前寄り × フラット)**: 打点が高く(≥1.7m)前寄り
  (ネットから 8.5m 以内)でフラットを打つと**スマッシュ**になり、初速が
  大幅に上がり(基準 42 m/s、チャージでさらに +最大45%)上から叩き込む。
  品質低下の影響も小さい。→ 高く弾む球や短いロブを前で仕留める決め球。
  ベースライン深くでは(前寄りでないため)成立せず、ただの高打点フラット。
- **低い打点でのパワーはアウト/ネットになりやすい**: 低い打点(lev<0)で
  強打(チャージ/クリーンヒット)すると、持ち上げて入れる必要があるのに
  パワーが乗るため**深さがオーバーして高確率でアウト**、フラットだと逆に
  ネットにも掛かりやすい。前寄りだとこのリスクはさらに増幅(後ろが短い)。
  → 低い球は無理に叩かず、トップスピン/スライスで安全に返すのが正解。
- **高い打点のトップスピンは角度をつけられる**: 高い打点(lev>0)のトップスピンは
  横の狙い幅が最大 +85% 広がり、回転も増えて沈むため、**鋭いクロスの
  アングルショット**が決まる(低い打点では角度をつけにくい)。
  → 高く跳ねた球を高い打点で捉えてオープンコートを作る。
- **前寄りフラットの深さリスク**: 前に詰めて低〜中打点でフラットを強打すると、
  後ろのコートが短いためアウトしやすい。前で安全に攻めるには高い打点(叩く)か
  角度・緩急が必要。
- **相手球の勢い(球威)**:
  - **カウンター/リダイレクト**: 速い球はフラット/スライスで合わせると相手の
    勢いを利用でき、自分の労力以上の速球で返る(球速の約30%上乗せ。
    トップスピンは約12%)。速い展開での「合わせる」技術。
  - **速球の制御難**: `vIn` が 17 m/s を超えると、体勢が悪い(品質が低い)ほど
    狙いが乱れる。速い球を走らされながら強打すると自滅しやすい。
  - **緩急の吸収難**: ドロップ/ロブのタッチショットは速い球からは打ちにくく
    (誤差増)、緩い球から打つのが成功率が高い。

→ 設計意図: これらは加算的・連続的に効くため、通常のベースラインラリー
(中くらいの打点・中庸な球威)はほぼ従来どおり安定するが、**極端な状況
(ネット際・高い/低い打点・速い球・走らされ)では結果が大きく変わる**。
「いつ前に出るか」「高い球をどう使うか」「速い球を合わせるか凌ぐか」という
判断が常に発生する。

### 4.6 速いボールの返球(ミート/差し込まれ)

スマッシュのような速いボールは、ボールが届く位置にいても**芯で捉えてミート
するのが難しい**。準備(チャージ)が浅いと差し込まれ、しっかり返せずに
**山なりの弱い返球(チャンスボール)**になる。これによりスマッシュ等の
強打に戦術的価値が生まれる(強く打つほど相手の返球が甘くなりやすい)。

現実のテニスのセオリーに基づくショット種ごとの「速球への強さ」:

- **スライス(ブロック)が最も強い**: 短いテイクバックで球威を利用して
  返せる。速いサーブのリターンでプロが多用する基本。低く滑って返る。
- **フラットは中間**: 球威を合わせて返せるが、タイミングがシビア。
- **トップスピンは最も弱い**: フルスイングと精密なタイミングが必要なため、
  速球に差し込まれると最も崩れやすい(ただし**十分に準備=チャージ**できれば
  強烈な返球も可能)。
- ロブ/ドロップのタッチショットは速球からは最も難しい(§4.5 とも整合)。

差し込まれ度 `mishit`(0〜1)は「相手球速がしきい値(26 m/s)をどれだけ超えたか
× ショット種の弱さ × (1−チャージによる準備) × 体勢(品質)」で決まる。
mishit が大きいほど、返球は **遅く・高く・浅く**(山なりのsitter)なり、
回転も失う。チャージで準備し、スライスで合わせれば速球も deep に返せるが、
不用意なトップスピンの当てるだけの返球は浮いて絶好球になる。

→ 戦術: 高い球を前で**スマッシュ**(§4.5)で叩き込めば、相手は差し込まれて
甘い返球になりやすく、次球で決められる。逆に、相手の強打を受けるときは
**早めにチャージしてスライスでブロック**するのが安全。通常のラリー球速
(〜25 m/s)では差し込まれは起きず、従来どおり打ち合える。

### 4.7 ネットプレー(ボレー)

ネット際(前寄り)でバウンド前のボールを**フラット/スライスで捉えるとボレー**になる。
振り抜かない**ブロック/パンチ**なので、威力は控えめだが**狙いは正確**、チャージしても
効きは小さい(溜め打ちの球種ではない)。前に詰めて甘い球を**正確に決める**ための手段で、
ベースラインからの強打(§4.5 の前寄りフラットはアウトしやすい)とは別物。
- 高い打点(≥1.7m)の前寄りフラットは**スマッシュ**(§4.5)が優先。
- ボレーヤーに対しては**ロブ**(頭上を抜く)や**パッシング**(サイドを抜く)が有効
   — ネットに出るリスクとリターンの読み合いになる。AI もペルソナにより前へ詰める(§7.1)。

## 5. サーブ

### 5.1 サーブの種類

サーブは方向(A/D)に加え、**種類を選べる**(サーブ前に J/K/L で選択。
デフォルトはフラット)。現実のテニスの3大サーブを再現する:

| 種類 | キー | 速度 | スピン | バウンド | 特徴・戦術 |
|---|---|---|---|---|---|
| フラット | J | 最速 | ほぼ無回転 | 低く直進 | 最速だがマージンが小さくフォルトしやすい。1stの決め球 |
| スライス | L | 中 | サイドスピン | 低く滑り横へ曲がる | 相手をワイドに追い出しコートを開ける。安全め |
| キック | K | 遅 | 重い順回転+横 | 高く弾んで跳ね上がる | ネット越えマージンが大きく安全。高く弾んで相手を後方/上へ押す。2ndの主軸 |

→ 戦術: 1stはフラット/スライスで攻め、フォルトしたら2ndはキックで安全に
入れる、という現実の駆け引きを再現。キックサーブは高く弾むため、相手は
高い打点で returns することになり(§4.5)、差し込まれや体勢の崩れを誘える。

### 5.2 サーブの操作と威力

1. サーブサイドの定位置に配置された後、**サーブを打つサイドの範囲内で
   立ち位置を移動できる**(x はセンターマーク±0.25m からシングルスライン、
   z はベースライン後方 0.2〜2.5m)。ワイドに寄って角度をつけるなどの
   駆け引きが可能。`Space` 押下でパワーメーターが往復開始
   (0→1→0、周期 1.2 秒の三角波)。メーター中は移動できない。
2. 離した瞬間の値 p が威力。A/D でボックス内の左右コースを選択。
3. 判定:
   - `p ∈ [0.70, 0.88]` — スイートゾーン。速く正確なサーブ。
   - `p > 0.88` — オーバーパワー。速いが誤差大、フォルト率高。
   - `p < 0.70` — 威力・誤差ともに p に比例(安全だが遅い)。
4. フォルト時の2ndサーブも操作は同じ。ダブルフォルトは失点。

**球速**: 初速は `SERVE_SPEED_MIN(30)..MAX(56)` を power で写像し、種別倍率(flat=1.0/
slice=0.9/kick=0.8)とペルソナの `serveSpeedMul`(serve 能力で最大 ~1.12)を乗算する。
ビッグサーバーのトップサーブは ~62 m/s(≈226 km/h)に達し、**コースを読めないと返球が苦しい**。

**サーブ後の硬直(リスク)**: サーブ直後、`SERVE_RECOVERY_MIN + SERVE_RECOVERY_GAIN·power` 秒間は
移動が大きく低下する(`SWING_LOCK_MOVE_FACTOR`)。**強打ほど硬直が長い**(トップサーブで ~0.75 秒)。
端から最速サーブを打つと**戻れず、良いコースにリターンされるとリターンエース**になりやすい。

→ 戦略: 1stは強気・2ndは安全という駆け引きに加え、**「速く・端から」打つほど自分の隙も大きい**という
リスクが乗る。常にトップスピードのワイドサーブを撃つのではなく、球速・コース・体勢復帰のバランスを取る。

## 6. スタミナ

**「強い行動(強打・スプリント)のクールダウン制」**(詳細式は ARCHITECTURE §6.5):

- **強い行動**を起こすたびにスタミナを消費し、同時に回復を `STAMINA_COOLDOWN`(2.5 秒)だけ停止する。
  最後の強い行動からクールダウンが経過して初めて `STAMINA_REGEN`(6/s)の回復が再開する。
  クールダウンはラリー1往復程度に設定してあり、毎回強打するとクールダウンが切れる前に次の強打が来て確実に減っていく。
- **歩行移動・繋ぎ・タッチ・セーフティ打球・弱いチャージ**は「強い行動」ではなく**無料(回復継続)**。
  消費ゼロで待機回復が続くので、繋ぎ中はスタミナが回復する。
- **強打(チャージショット)**: 正規化チャージ `c = charge/CHARGE_MAX ∈ [0,1]`。
  `c ≥ CHARGE_STRONG_THRESHOLD`(0.35)のみ「強打」= 消費&回復停止の対象。弱打は無料。
  コストは `chargeShotCost(charge) = STRONG_SHOT_COST_MAX·(c−閾値)/(1−閾値)`(閾値で0、フルチャージで 10)。
  **ペルソナによる消費差はなし**。誰が打っても同じだけ減る。
- **スプリント**: スプリント中 `STAMINA_SPRINT_DRAIN`(15/s)を消費(移動時間に比例)。**ペルソナ差なし**。
- **サーブ**: 強い行動扱い。`SERVE_STAMINA_MAX·power`(= 8×power)を消費し、クールダウンをリフレッシュ。
  サーブ自体は常に可能(スタミナが少なくてもサーブは打てる)。
- **基本ストック**: 最大チャージ強打 10 回分 = `STAMINA_MAX`(100)。**ペルソナの stamina レーティング
  r はストック量(上限)のみ**を増減: `staminaMaxMul = 0.7 + 0.1·r`
  (r=1→0.8=80=8発 / r=3→1.0=100=10発 / r=5→1.2=120=12発)。
  消費・回復・クールダウンは全員共通(旧モデルの「上限+消費+回復」三重取りによる満タン張り付きを解消)。
- **スタミナ切れペナルティ = 能力ゲート**(品質は下げない):
  - **強打不可**: 残量 < `CHARGE_ENABLE_PCT`(effStock の 15%)ではチャージが溜まらず通常打のみ。
    この閾値は最大強打1発分(10)を上回るので、チャージを始められた=最後まで打ち切れることを保証する。
  - **スプリント不可(ヒステリシス)**: 残量 ≤ `SPRINT_STOP_PCT`(3%)でスプリント強制停止、
    `SPRINT_RESUME_PCT`(18%)を超えるまで再開不可。境界での ON/OFF 振動を防ぐ。
- **ポイント間回復**: `STAMINA_POINT_RECOVERY`(40)を加算(精神力 `clutchRecoveryMul` で増減)。
  ポイント間にクールダウンとスプリントロックは解除。長丁場では取り切れず蓄積疲労が残る。
- **精神力(隠し)**: ペルソナごとの mental(1..5)が、プレッシャー局面でのポイント間回復に効く。
  勝負強いタイプは回復が速く重圧で粘り、脆いタイプは競り合いで失速する。
- HUD ではプレイヤーの脇に控えめな**円形ゲージ**(緑→黄→赤・数字なし・半透明)で表示し、
  低下時は**発汗**(`STAMINA_SWEAT_START`=50% 未満)で体感的に伝える(相手のゲージは非表示)。
  強打・スプリントのたびに確実に減り、繋ぎや静止で回復する。

→ 戦略: 毎回強打を連発すると、クールダウンが切れる前に次の打球が来てスタミナが確実に減っていく。
チャンスだけ強打し、繋ぎはタッチ・トップスピンで凌ぐ配分が長期的に効く。
スプリントも有限——走らされない配球・オープンコートを作る判断が体力温存につながる。

### 6.1 プレッシャー(重要局面)

スコア状況から重圧 `p`(0..1)を算出し、精神力と掛け合わせる:
マッチポイント ≈ 1.0、ブレークポイント ≈ 0.8、ゲームポイント ≈ 0.7、デュース ≈ 0.4、
30-30 ≈ 0.25、その他 0。高 mental は重圧下で消耗が減り、低 mental は増える。
さらに**重圧下のショット品質**も mental で変動: 低 mental は重圧で精度が落ち(choke)、
高 mental はむしろ安定/微強化(clutch)する。

### 6.2 モメンタム(勢い)

連続得点で**勢い**(momentum, −1..+1)が増減する。3点連取で最大。波に乗っている側は
ショット品質が微増、連続失点で崩れている側は微減する。HUD に勢いインジケータ(🔥)を表示。
→ 流れを掴んだら一気に畳みかけ、悪い流れは断ち切る(緩急・確実な繋ぎで止める)駆け引き。

### 6.3 オープンコート

相手を左右に走らせてセンターから外せば、AI は予測点へ正直に動くため戻りが間に合わず、
空いた側を突けば決まる(配球 §4.3 の成果)。走らせ距離はマッチ後スタッツに集計される。
※ 床の光るハイライト演出は装飾過剰のため**無効化**(`main.ts` の `OPEN_COURT_ENABLED`)。

## 7. AI

### 7.1 行動モデル

- ラリー中はボールの**着地予測点**へ移動(反応遅延後)。
  自分が打たないときはホームポジション(ベースライン中央寄り)へ戻る。
- **戦術スタンス(ベースライン / ネット)**: 相手の打球ごとに「後ろで打ち合う(baseline)」か
  「前へ詰めてボレー(net)」かを判断する。**既定はベースラインのラリー。短い球(チャンス)が
  来たときだけ前へ詰める**。判断は **チャンス量**(着地がネット寄りなら+/速い球は−)が、
  **ネット志向(ペルソナ性格)で決まる必要量**を超えたら net。ネット型ほど小さなチャンスでも
  前へ出て、グラインダーはほぼ後ろで打ち合う。→ 積極派でも普段はラリーし、時々チャンスを見て前へ。
  - **baseline**: 着地点より**深く(ネットから遠く)下がって構え**、バウンド後の上がり際〜頂点で打つ。
    着地点ちょうどに立って低いバウンドを弱く返す挙動を避け、適正な高さ・威力で打ち合う。
  - **net**: 着地点より**前(ネット寄り)に出て**、バウンド前=空中でボレー気味に捉える(ネット際まで詰めすぎない)。
    前で高い打点を取れれば決め球(スマッシュ/強打)につながる。詰めた後はベースラインまで戻らず
    前目で構え直す(毎球の往復による消耗を避ける)。
  - ネット志向は**ペルソナの性格で調整**(サーブ&ボレー/オールコート型は前へ、グラインダー/
    カウンター型は後ろで粘る)。同じ難易度でもペルソナで戦い方が変わる。
- **アウトの見送り**: 相手の打球の着地予測が自陣コート外(ベースライン奥/サイド外)の
  ときは打たずに見送る。明らかにアウト(ライン外 0.6m 超)なら高確率で見送り、
  きわどいもの(ライン際)は確率的に見送る/打つ。難易度が高いほど見送り精度が高い
  (Hard はほぼ確実に明らかなアウトを見送る)。コート内に入る球は必ず打つ。
  見送ると判断した球は追わずホームへ戻る。
- **サーブのリターン位置取り**: レシーブ側のとき、プレイヤーのサーブ立ち位置を読んで
  リターンしやすい場所へ移動する。サーバーから打てる両極(ワイド/センターT)の
  サーブ軌道を構え位置まで延長した二等分点に立つ(サーバーがワイドに寄るほど
  レシーバーもワイドへ寄る)。難易度が高いほど的確に最適点へ寄せ、低いほど汎用の
  定位置のまま(賢い相手ほどサーブが返されやすくなる)。
- ショット選択は以下を重み付けで評価:
  - **オープンコート**: プレイヤー位置から遠いサイドを狙う(基本)
  - **体勢**: 走らされているときは安全なショット(トップスピン/スライス/ロブ)
  - **プレイヤーが前に出たら**ロブ/パッシング(サイド強打)
  - **プレイヤーが深く下がったら**ドロップショット
- AI にも品質システムとスタミナが同じルールで適用される。

### 7.2 難易度(タイトル画面で選択)

| パラメータ | Easy | Normal | Hard |
|---|---|---|---|
| 反応遅延 | 0.42秒 | 0.28秒 | 0.16秒 |
| 移動速度係数 | 0.82 | 0.95 | 1.05 |
| 狙いの誤差(追加ノイズ) | 大 | 中 | 小 |
| 強気な配球の頻度 | 低 | 中 | 高 |
| 凡ミス率(時々ある明らかなミス) | 8% | 4% | 1.5% |
| 明らかなアウトの見送り率 | 82% | 92% | 98% |
| きわどいアウトの見送り率 | 35% | 45% | 55% |
| リターン位置取りの的確さ | 0.25 | 0.6 | 1.0 |

→ どの難易度でも「左右に振ってから逆を突く」「前後の揺さぶり」という
**人間的な攻略法が通る**ように、AI は予測点ベースで動き先読みはしない。

## 8. カメラ・演出

- プレイヤー後方上空からの追従カメラ(位置・注視点ともに緩い lerp)。
  ボールとプレイヤーの中間点を注視。
- **ボールの視認性**(最優先。ボール自体は実寸の 1.35 倍に留め、補助表現で追いやすくする):
  - 発光ハロー(加算合成の billboard)でボールを目立たせる
  - 長く太い残像トレイル(速度が速いほど顕著)
  - **着地予測マーカー**: 飛行中のボールの着地点にリングを表示(脈動アニメ)
  - ボール真下のグラウンドマーカー(擬似丸影を強調)
  - バウンド時にリング状のダストエフェクト
- **プレイヤーモデル**: 頭・胴体・両腕・両脚を持つ人型。**利き手にラケット**を持ち
  (左利きのナダウは左手持ち=モデルを鏡像化)、フォアハンド/バックハンドのスイングが見分けられる。
  向きは移動・待機中は**ネット側(コート中央)を向き**、横移動で左右に傾く(左右の方向転換は
  ネット側を通り、後ろを向いて回らない)。**地上ストローク(フォア/バック)では体を横に向けて
  打つ**(テイクバックでコイル→振り抜きで正面へ。side-on でリアル)。**スマッシュ/サーブは
  正面向き**のオーバーヘッドモーション。
  チャージ中はボール側へテイクバック(チャージ量で引きの深さが変わる)。
  **打点が高い(`SMASH_MOTION_HEIGHT` 以上)とき、およびサーブは、横振りでなく
  頭上から叩くオーバーヘッド(スマッシュ)モーション**で振る(`PlayerView.swingKind='smash'`)。
- ポイント決着時: スロー演出はなし。バナー表示(例「15 - 30」「Game!」)
  → 1.8 秒後に次のサーブへ。

## 10. オーディオ

バウンド音、ネット音、ポイント決着の歓声(フィルタ付きノイズ)、UI クリック音は
WebAudio 合成(外部ファイルなし)。**打球音のみ実録音サンプルを使用**。初回操作で
AudioContext を resume。

**打球音**(`audio/sfx.ts` `playHit(shot, opts)`):
- **実録音サンプル(効果音ラボ「テニスラケットで打つ」)を使用**。ビルドに同梱した自前ホストの
  音源を `decodeAudioData` で読み込み、`BufferSource` で再生する。合成では「電子的」で打球感が
  出なかったため実音源に切り替えた経緯(出典・規約は `README.md` / `src/audio/samples/CREDITS.md`)。
- ショット種別の描き分けは **playbackRate(音程)・音量・ステレオパン・フィルタ**で行う
  (定数 `HIT_SAMPLE_*`): フラット=基準、スライス=高め、ロブ/ドロップ=低め、サーブ=重く低め、
  差し込まれ(mishit)=レート低下+ローパスで鈍い「コツッ」。強打ほどわずかに高く鋭く、
  打点 x でパン、毎打の微ピッチ揺らぎで反復感を消し、複数サンプルをラウンドロビン。
- **サンプル未ロード/取得失敗時は WebAudio 合成版(`playHitSynth`)へフォールバック**(下記 5 レイヤ)。
  音源が無くてもゲームは動作する。

**(フォールバック)合成での打球音の作り分け**(定数 `HIT_SOUND_PARAMS`/`SFX_*`)。
実テニス打球音の音響特性(調査: 基音/ボディ 100〜1800Hz・倍音 1800〜2800Hz、衝撃はごく短く
鋭い。深み=低域 / クリスプさ=高域 + 鋭いアタックで "POCK!")を 5 レイヤ合成で再現する:
- **① ボディ(pock)**: 三角波を高い周波数から `bodyHz` へ素早く降下させる。音程感のある芯=
  打球のパンチ。打球音の主役。
- **② クラック(アタック)**: 明るいハイパスノイズの鋭い立ち上がり。"パッ" の抜け・クリスプ感。
- **③ シマー**: 1.8〜2.8kHz 帯のバンドパスノイズ。高域の煌めき(クリスプさの素)。
- **④ 弦面リング**: `bodyHz` の高Qバンドパスノイズの短い余韻。`sweep` で中心周波数を
  上昇(トップスピン=擦り上げ)/下降(スライス=切り)。
- **⑤ ブラシ/擦過ノイズ**: 回転種で擦過感を付加。
- ショット別の音色(§7.4): フラット=鋭い「パッコーン」、トップスピン=擦って弾く、
  スライス=薄く滑る、ロブ=柔らかい、ドロップ=触れるだけ。
- 爽快感の共通要素: **球威(intensity)で明るく鋭く**(ボディ/シマー/クラックの周波数を上げる)、
  打点 x で**ステレオ定位**、毎打わずかに**微ピッチ揺らぎ**(反復感を消す)、
  **手続き生成 IR の薄い残響**(ConvolverNode)。サーブはフラット増強、ジャストは澄んだ
  ベル倍音、差し込まれ(mishit)はボディを鈍く・ノイズ多めで詰まった「コツッ」。

## 11. 戦略性の設計意図(まとめ)

1. **配球**: 5種のショット × 左右前後の打ち分けで相手を動かし、オープンコートを作る。
2. **リスク管理**: 品質システムにより「体勢が悪いのに強打」は自滅する。
   ライン際を狙うか安全に返すかの判断が常に発生する。
3. **体力管理**: 強打・スプリントは有限のストックを消費する。連発するとクールダウン中に回復できず確実に枯れる。繋ぎ・歩行移動は無料なので、チャンスだけ強打する配分が長期的に効く。
4. **サーブの駆け引き**: パワーメーターによる 1st/2nd のリスク選択。
5. **AI の隙**: AI は正直に予測点へ走るので、振り回し・逆を突く・前後の
   揺さぶりという現実のテニスのセオリーがそのまま攻略法になる。

## 12. プレイヤータイプ(ペルソナ)

プレイ開始時に**自分**と**相手**のプレイヤータイプ(ペルソナ)をそれぞれ選択する
(「おまかせ」=ランダムも可)。難易度(Easy/Normal/Hard)とは直交し、難易度=相手AIの
総合的な上手さ、ペルソナ=プレイスタイルと能力の配分(誰にも武器と弱点がある)。

### 12.1 能力6軸

各ペルソナは6軸・各1〜5段階で表され、タイトル画面でレーダーチャート表示される:

| 軸 | ゲーム内の効果(強いほど) |
|---|---|
| サーブ | サーブ初速↑・フォルト率↓ |
| パワー | ストローク初速↑・チャージ威力↑ |
| スピン/安定 | 狙い誤差↓・ネット越えマージン↑・速球に差し込まれにくい |
| スピード | 移動速度↑・リーチ(届く範囲)↑ |
| スタミナ | ストック量(effStock)↑ = 強打・スプリントできる回数↑。消費・回復速度は全員共通 |
| 技巧 | ドロップ/ロブ/スライス/アングルの誤差↓・速球リターンが上手い |

能力値→倍率の導出は ARCHITECTURE §6.5。倍率は既存定数に掛けるだけで、新システムは足さない。

### 12.2 6タイプ

| タイプ | 型 | 武器 / 弱点 |
|---|---|---|
| ピート・サンブラント | ビッグサーバー | 一撃サーブと前の決定力 / 粘りに弱い |
| アンドレ・アガチ | リターナー | 速球を叩き返すリターン / ネット・タッチが苦手 |
| ノヴァ・ジョコヴィン | カウンター | 最高の安定と粘り・スタミナ / 決定力に欠ける |
| ケイ・ニシゴオリ | スピード/技巧 | 機動力と多彩なタッチ / スタミナが弱点 |
| ラファ・ナダウ | グラインダー | 重い順回転と無尽蔵スタミナ・左利き / タッチ・サーブは平凡 |
| ロジャー・フェデルン | 攻撃的万能 | 穴のない攻撃 / 長い我慢比べで消耗 |

おおまかな三すくみ(攻撃型↔グラインダー↔スピード/技巧型)で選択に意味を持たせる。

### 12.3 身体・外見

各ペルソナは身長・体格・利き手(身体)と髪型・袖・アクセント色(外見)を持つ。
3Dモデルは **チームカラー(自=青/相手=赤)で陣営**を、**シルエット(身長・体格・髪・袖)+
アクセント色でペルソナ**を二重に識別できる。左利き(ナダウ)はフォア/バックが左右反転する。

### 12.4 精神力(隠しパラメータ)

レーダーに出さない隠し `mental`(1..5)を各ペルソナが持ち、プレッシャー局面(§6.1)での
消耗と時間回復に効く(可視の stamina 軸とは別レイヤー)。

| ペルソナ | mental | キャラ付け |
|---|---|---|
| サンブラント | 4 | ビッグマッチに強い |
| アガチ | 3 | ムラがあり競り合いで揺れる |
| ジョコヴィン | 5 | 鉄のメンタル。重圧で逆に強く回復も早い |
| ニシゴオリ | 2 | 終盤に失速・競り合いで消耗(先行逃げ切り型) |
| ナダウ | 5 | 不屈。長期戦・重圧に滅法強い |
| フェデルン | 4 | 沈着。重要局面でも崩れにくい |

→ ニシゴオリは速さ・技巧で序盤リードするが長いラリーとブレークポイントの連続で失速しやすく、
ジョコヴィン/ナダウは競るほど強い。「短期で決めるか/粘って削るか」の駆け引きになる。

## 13. コートサーフェス

タイトル画面でコートを選べる。バウンドと球速が変わり、ペルソナ相性に環境変数が加わる:

| サーフェス | 球速 | バウンド | 有利なタイプ |
|---|---|---|---|
| ハード | 中 | 中 | 万能(基準) |
| クレー | 遅い | 高く跳ねる | グラインダー/カウンター(ナダウ・ジョコヴィン)。高い打点を強要でき、走って拾える |
| グラス | 速い | 低く滑る | ビッグサーバー/攻撃型(サンブラント・フェデルン)。サーブ&ボレーが効く |

実装は反発(REST)・水平摩擦(BOUNCE_FRICTION)・空気抵抗(KD)をサーフェス係数で
スケールするだけ(`SURFACE_PARAMS` / `setSurface`)。コート面とラインの色も変わる。

## 14. 練習モード(ボールマシン)

スタートメニューで **Mode = Practice** を選ぶと、対戦の代わりに**ボールマシンが同じ種類の球を
繰り返し打ち出す**練習モードになる(スコアなし)。ジャストミートのタイミング習得・検証用。

- **ボール種類**(Ball): フラット / トップスピン / スライス / ロブ / ランダム。
- **コース**(Course): コート前(ネット寄り)/ コート後(ベースライン寄り)/ ランダム。横方向は
  小さくばらつく(`PRACTICE_SPREAD_X`)。
- マシンは相手ベースライン中央から `solveShot` で選択種類・コースへ球出し。決着(2バウンド/場外)
  後 `PRACTICE_FEED_DELAY` 秒で次球。プレイヤーは打ち返すだけ(相手 AI は立っているだけ)。
- **成績表示**(HUD 左上): **ジャスト**(芯で離して成功)/ **通常**(ジャストでない打球)/
  **セーフティ**(離さず自動で打たれた)の累計回数。`?debug` では毎打の判定ログも出る。

→ 「離すタイミングがどれだけ芯に合っているか」を、球種・コースを固定して集中的に確認できる。
